# はじめに
[以前の記事](https://tok41.github.io/python-WebCamera.html)で
Webカメラの画像から特定の色だけを抜き出すフィルタを実装しました。
この仕組みを拡張すれば、
画像の中で指定の色を持つ領域（物体）を認識することが容易に実現できます。

しかしながら、
上記の仕組みはあくまでカメラキャプチャした静止画像に対して行うものです。
静止画に対する処理を連続することで動画像に対して物体の領域認識ができているように見えますが、
それは人間の感覚のためであって、
コンピュータは毎回独立にキャプチャされる画像に対して独立に処理をしているにすぎません。

動画中の特定の物体（ここでは色）がどのように動いているのかを
コンピュータが認識するためには、
前回のフレーム画像における物体の位置が
今回キャプチャした画像の中でどのように変化しているのか、
時系列でどのような関係性を持っているのかを明示的に推定する必要があります。

これは、
流行りの深層学習での物体検知を単純に使った場合でも同様で、
物体認識と追跡は別に考える必要があります。

（深層学習を利用したトラッキング技術は存在するので、
あくまでも物体検知を単純に使ってもトラッキングはできないよということです）


# アプローチ

パーティクルフィルタを使います。


# パーティクルフィルタ
状態が遷移して、遷移した状態で観測が得られるというモデルを考えます。

## 状態遷移モデル
ここでは簡単のために、前の位置からランダムに動くと仮定したモデルを作成します。

## 観測モデル
今回は、色が指定の色に近いほど尤度が高くなるように観測モデルを設計します。
